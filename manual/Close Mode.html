    Close mode

    Other than opening a state, we can even close it.

    Normally , we can do it manually by calling a classic Surfer action by selecting Close State as a mode.

    But automatic is better than manually.

    The Close Mode of a state (that we can found after choosing State as Element type), it automatically closes all the states specified without closing manually all of them.

    Example :

    We have two siblings Tab Panels, Tab1 and Tab2.

    When one of them is open, the other one should be closed. By doing it manually, we would have to call 4 actions : OpenState Tab1 and CloseState Tab2 , or OpenState Tab2 and CloseState Tab1.

    By using Close Mode, if we choose Siblings as mode, the moment we call OpenState Tab1, the CloseState Tab2 action will be automatically triggered and viceversa.

    Runtime way

    [SerializeField]
RectTransform _panel = default;

private void Awake()
{

    SUElementData stateData = new SUElementData();


    //this is how you create the state data related to a Canvas/UGUI panel 
    stateData.SetUpState(_panel.gameObject,"MyStateName",0);

    //this is how you create the state data related to a UIToolkit panel
    stateData.SetUpState(_myVisualElementPanel,gameObject,"MyStateName",0);


    //here we set the closeMode
    stateData.SetCloseMode(SUElementData.StateCloseMode_ID.Siblings);


    //here we register to the state enter and exit event of the state
    SurferManager.I.RegisterStateEnter(this,"MyStateName");
    SurferManager.I.RegisterStateExit(this,"MyStateName");

}

//implementation of interface ISUStateEnterHandler
void OnSUStateEnter(SUStateEventData eventInfo)
{

}

//implementation of interface ISUStateExitHandler
void OnSUStateExit(SUStateEventData eventInfo)
{

}
