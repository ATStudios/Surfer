    Tutorial Click Constraint

    In order to create a UI panel where only a specific button can be clicked (like if the user must follow a tutorial and therefore is obliged to click ONLY that button), we must simply choose the reaction UIGenerics/EnableClickConstraint .


    Here's a simple example :

    • Create a simple Menu with 3 buttons



    • Then, on the Menu gameObject, add the component SUElementCanvas and set it up as a state with the name “Menu"



    • Now we decide that this Menu State will have two behaviors : NORMAL behavior with all buttons working (State Version = 2) and TUTORIAL behavior with only the play button working so that the player is forced to click only that button (State Version = 1)



    • So, in the same component as above, we will add two events : MyStateEnter (Version : 1) with EnableClickConstraint as reaction and Play button as object of the reaction; and MyStateEnter (Version : 2) with DisableClickConstraint as reaction and Play button as object of the reaction



    • Then , create an empty object and add a SUElementCanvas component to it and setup the opening of the Menu State on the MySceneLoaded event : after 0 seconds you will load the version 1 (only play button clickable) and then after 4 seconds you will open version 2 (all buttons clickable). In this way you pass from a Menu where you oblige the player to click only the play button to a normal Menu where everything is usable






    Now, hit play. After 0 seconds you will see only the Play button clickable :



    And after 4 seconds, you will now see the Menu panel with all the buttons clickable , because it exited from its "Tutorial" behavior

    INFO : This reactions use CanvasGroupcomponents in order to work. So , if the expected constraint is not working, it is probably due to a conflict of the components with your current UI logic.
