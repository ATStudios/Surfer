    Input Icons



    Let's see how to automatically manage the Input Icons based on the platform.

    Platform Icons setup.

    First, let's setup the icons; for this tutorial, download the UI-Icons available at this link.

    Then, import the folder into the project and convert all the icons to Sprite type in the inspector.

    Then, go to Unity->Preferences->Surfer->Input Icons Managers. For this example we will manage Desktop and Mobile platforms so: next to New Platform let's select Mobile and then click Add. Then, let's do the same for Desktop.

    Under Choose platform Icons to edit , let's choose Mobile ; then click + next to Sprites.

    In the Popup, let's give the name action and let's choose a random sprite from the ones downloaded before. The click Save.

    Under Choose platform Icons to edit , let's choose Desktop now ; then click Edit next to action, and in the popup let's choose a sprite different than the previous one.

    Canvas

    To use a platform-input icon just setup, let's first add a UI Canvas object with an Image component added to it. Then let's add the component SUInputIcon to it and select from the dropdown Sprite the choice action.

    Now, if you hit play , if the current selected BuildTarget is a Mobile one you will see the Mobile input icon chosen, otherwise if the target is a Desktop one you will see the Desktop icon chosen.

    UIToolkit/Runtime

    To get the icon just setup at runtime, we must simply do this :

    //to get the sprite
var sprite = SUInputIconsManager.I.GetIconSpriteFromName("action");
//to get the sprite (resources)
var resSprite = SUInputIconsManager.I.GetIconResSpriteFromName("action");

    Then, to use it for a VisualElement, we do this:

    img.style.backgroundImage = new StyleBackground(sprite);
